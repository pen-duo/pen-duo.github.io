git远程仓库

前言:和我一起开始git远程仓库的学习吧

git远程仓库

1.利用ssh sky方式本地连接远程

如果有本地仓库:

ssh-keygen -t rsa -b 4096 -C “your_email@example.com“(新建项目中有代码直接赋值)
一直回车,直到没有提示
cat ~/.ssh/id-rsa.pub得到公钥内容,黏贴到github设置中的ssh-key中.(如果成功会出现一段很长的公钥代码)
测试:ssh -T git@github.com (如果成功,出现名字以及带success字眼的句子)

如果没有本地仓库

git init
git add xxx
git commit -v
先创建一个本地仓库在进行上述操作。
不想上传的文件可以放在.ignore文件中。

2.上传与下载

上传

与远程仓库链接好后就可以进行上传以及下载了

git remote add origin git@xxxxx(上传到远程的master分支)
在本地添加远程仓库
origin是远程仓库,可以换,建议不换
不要使用https://,每次都需要密码

git push -u origin master
推送本master分支到远程origin的master分支
如果提示一个git pull一下,就git pull一下
git是先把远程分支合并到本地对于的分支
如果远程分支没有更新过就忽略git pull
-u origin master的意思是设置上游分支
之后就不用设置上游分支了,直接git pull;git push

如何上传其他分支
git push origin x:x

可以指定版本提交
git reset –hard XXXXXX指定版本的前六位
git log/git reflog可以查看指点版本
注意:在切换版本之前一定要确定所有主要的内容都被git commit了,否则没有被提交的会被删除

下载

git clone git@xxxx[目标路径].git
如果是不同路径,要写上传的新的ssh key(一机一key)
cd 目标路径
git add//git commit//[git pull]//git push四连操作

git clone git@xxx.git yyy
会在本地创建yyy目录
cd进yyy目录

git clone git@xxxx[目标路径].git .
最后一个字符是.,注意有空格
不会创建新目录
使用当前目录容纳代码和.git
当前目录最好是空目录

如何下载某个分支
先下载整个仓库,然后git checkout分支名

3.切换远程仓库

git remote -v查看当前连接的远程仓库
git remote set-url origin 项目ssh地址

4.切换到制定版本

git reser –hard
经验之谈:当新建了一个远程仓库并且需要向此远程仓库添加最近的项目时记得新建一个本地仓库git init。
