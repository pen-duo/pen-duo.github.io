前言:和我一起开始git本地仓库的学习吧

git本地仓库

1.git是什么

git是开启github的钥匙
git只是一个命令而已,Bash命令行中有很多命令
git可以解决的问题:版本控制。

2.git config配置git

注意：上面的英文名和邮箱跟 GitHub 没有关系。
可以跟 GitHub 的用户名和邮箱保持一致，
也可以不一致。我的是一致的。

注意：你需要保证 code 是可以直接在命令行执行的。
如果不能执行，你需要安装 VSCode 并配置 PATH。
我的 PATH 里添加的内容是
C:\Users\Fang\AppData\Local\Programs\Microsoft VS Code\bin

3.git 常见命令

  • git init 初始化
    • git init会创建.git目录
  • git add 路径 标记要被提交的文件
  • .gitignore 一个文件 里面放着那些不要被提交的文件
  • git commit -m 字符串 提交并说明理由(字符串中如果有空格就用””包起来)
  • git commit -v 功能同上 不过会打开vscode文件写注释(推荐使用)
  • git log 查看历史创建版本以及注释 只会查看当前最新版本之前的所有版本
  • git reset –hard 版本前六位 返回指定版本
  • git reflog 查看全部历史版本(高级)
  • git status 状态信息
  • git status -sb 状态简写
  • git branch x 创建另外一个版本(虚的)
  • git checkout x/master 切换分支 ()
  • git branch 查看当前在那条线上
  • git merge 将一个分支合并到当前分支
    • 注意:如果合并时产生冲突
      • 解决步骤:
      1. 发现冲突(conflict)并使用git status查看哪个文件冲突了
      2. 解决冲突
        1. 依次打开文件
        2. 搜索====四个等号
        3. 在上下两部分中选择要保留的代码
        4. 可以只选择上面,也可以只选择下面
        5. 删除不用的代码,删除====,>>>>,<<<<这些标记
        6. git add对应文件
        7. 再次git status -sb ，解决下一个文件的冲突。
        8. 直到没有冲突,运行git commit(注意不需要选项)。
        9. git branch -d x删除无用的x 分支

一些细节:

  • 大部分时候,只需要使用git add.和git commit -v操作即可
  • 使用时git reset –hard 指定版本前六位之前必须确保所有重要代码都commit了,因为处于被add标识过但是没有commit的文件全部都会被删除
