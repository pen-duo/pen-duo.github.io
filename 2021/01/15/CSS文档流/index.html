CSS文档流

前言:和我一起开始CSS文档流的学习吧

CSS文档流

文档流

流动方向

• inline元素从左到右,一直到最右边才换行
• block元素从上到下,每一个都另起一行
• inline-block也是从左到右

宽度

• inline宽度为内部inline元素的和,不能用width指定
• block宽度为默认自动计算宽度,可用width指定
• inline-block宽度结合前两点的特点,可指定宽度。

高度

• inline高度为line-height间接指定,跟height无关(默认不为0)
• block高度由内部文档流指定,可以设置height (默认为0)

  • inline-block跟block类似,可以设置height

overflow溢出

当内容大于容器
• 当内容的宽度或高度大于容器,会溢出
• 可以overflow来设置是否显示滚动条
• auto是灵活设置
• scroll是永远显示hidden是永远隐藏溢出效果
• visible是永远显示溢出效果

脱离文档流

• float
• position:absoluate/fixed会导致脱离文档流

两种盒模型

CSS盒模型有两种盒模型
content-box内容盒模型 width = 内容宽度
border-box边框盒模型 width = 内容宽度 + padding + border

margin合并

那些情况会合并
• 父子合并
()[https://cdn.nlark.com/yuque/0/2020/png/1554318/1606740072235-97d5e83d-a4d4-4a54-839c-9e5f33b64b4d.png?x-oss-process=image%2Fresize%2Cw_1472]

parent的上外边距会和son的上外边距合并,取两者中较大的一个。
• 兄弟合并
()[https://cdn.nlark.com/yuque/0/2020/png/1554318/1606739802670-5bf1c4c9-594c-4341-ba44-34401205e71a.png?x-oss-process=image%2Fresize%2Cw_1472]
box1的下外边距和box2上外边距不会累加,而是会去两者中较大的。

如何阻止合并
• 父子合并用padding/border挡住
• 父子合并用overflow:hidden挡住
• 父子合并用display:flex挡住
• 第三种方式是给parent加overflow:hidden
• margin能合并就是因为父子的Margin之间没有其他的东西，如果加了border或者Padding就等于在父子的Margin中间加了东西，因此就无法合并margin了
• 兄弟合并用display:inline-block;
