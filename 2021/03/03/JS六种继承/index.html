JS六种继承

1.原型链继承

function Father(name){ this.name = name; } function Son(name){ this.name = name; } Son.prototype = new Father();

重点:让子类的原型等于父类的实例

特点:子类实例可以继承子类构造函数的属性,父类构造函数的属性,父类原型的属性。

缺点一:子类实例无法向父类构造函数传参。

缺点二:子类实例共享了父类构造函数的引用属性

2.构造函数继承

function Father(name){ this.name = name; } function Son(name){ this.name = name; Father.call(this); }

重点:在子类中利用call,apply方法引入父类构造函数。

特点:子类实例可以继承父类构造函数的属性,子类构造函数的属性,不会继承父类原型的属性。

优点一:子类实例之间独立。

优点二:创建子类实例，可以向父类构造函数传参数。

优点三:子类实例不共享父类构造函数的引用属性。

缺点一:只能继承父类构造函数的属性。

缺点二:子类实例，继承不了父类原型上的属性。

3,组合继承(组合原型链继承和借用构造函数继承)

function Father(name){ this.name = name; } function Son(name){ this.name = name; Father.call(this); } Son.prototype = new Father();

重点：结合了两种模式的优点，传参和复用

优点一:可以向父类传参

优点二:父类的实例方法定义在父类的原型上,实现方法复用

优点三:子类实例相互独立

缺点：调用了两次父类构造函数（耗内存），子类的构造函数会代替原型上的那个父类构造函数。

第一次Son.call(this);从父类拷贝一份父类实例属性，作为子类的实例属性，

第二次Son.prototype = new Father();创建父类实例作为子类原型，此时这个父类实例就又有了一份实例属性，但这份会被第一次拷贝来的实例属性屏蔽掉，所以多余。

4.原型继承

重点:创建一个空对象,指定空对象的原型为object的参数。

缺点一:所有实例都会继承原型上的属性。

缺点二:无法实现复用。

5.寄生继承

重点:就是给原型式继承外面套了个壳子

优点：没有创建自定义类型，因为只是套了个壳子返回对象（这个），这个函数顺理成章就成了创建的新对象

缺点：没用到原型，无法复用

6.寄生组合式继承（常用）

优缺点：这是一种完美的继承方式。

7.ES6中的类与对象

类抽象了对象的公共部分,它泛指某一大类

对象特指某一个,通过类实例化一个具体的对象

面向对象的思维特点:

1.抽取(抽象)对象共用的属性和行为组织(封装)成一个类。

2.对类进行实例化,获取类的对象。

(1)通过class关键字创建类,类名我们还是习惯性定义首字母大写。

(2)类里面有个constructor函数,可以接受传递过来的参数,同时返回实例对象。

(3)constructor函数只要new生成实例时,就会自动调用这个函数,如果我们不写这个函数,类也回自动生成这个函数。

(4)生成实例new不能省略

(5)最后注意语法规范,创建类 类后面不要加小括号,生成实例 类名后面加小括号,构造函数不需要加function.

(6)我们类里面所有的函数不需要写function

(7)多个函数方法之间不需要添加逗号分隔

创建类

类的继承

现实中的继承:子承父业,比如我们都继承了父亲的姓。

程序中的继承:子类可以继承父类的一些属性和方法

语法: 父类名 extends 子类名

super关键字用于访问和调用对象父类上的函数。

可以调用父类的构造函数,也可以调用父类的普通函数。

子类在构造函数中使用super,必须放到this前面(必须先调用父类的构造方法，在使用子类构造方法

)

继承中属性或方法的查找原则

1.继承中,如果实例化子类,输出一个方法,先看子类有没有这个方法,如果有就先执行子类的。

2.继承中,如果子类 没有,就去查找父类有没有,如果有,就去执行父类的这个方法

super调用父类构造函数

super调用父类普通函数

我们可以用extends使得子类继承父类的属性和方法,但是想调用父类的参数,需要用到super方法。

ES6中的类和对象三个注意点

1.在ES6中没有变量提升,所以必须先定义类,才能通过类实例化对象。

2.类里面的共有的属性和方法一定要加this调用

3.constructor里面的this指向实例对象,方法里面的this指向这个方法的调用者
